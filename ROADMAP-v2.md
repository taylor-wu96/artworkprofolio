## 〇、為什麼有 v2

[ROADMAP v1](ROADMAP.md) 已經把「骨架比肉身豐富」翻成行動並全數落地——放映 Grid、真 EXIF、image-forward OG、論述面板、Series 連續放映。**儀器齊全了。** 所以 v2 不再造零件。

v2 處理的是兩件**表面上不同、本質上同一**的事：

- **感性面**：觀者抱怨不出口、但身體感覺得到的——這座館的零件們**還不像同一個身體**。換頁是硬切、暗房之間沒有連續、三個影像入口讓人不知道該推哪扇門。
- **理性面**：創作者抱怨得出口的——**改 A 壞 B、做了卻只做一半、改了卻沒套用到**。

把它們放在一起看，會發現是同一個問題的兩面：

> **v2 要做的一體性工作（統一過場、連續的暗、跨頁轉場），剛好全部落在工程上最脆弱的那塊地——全域過場與 view-transition。沒有護欄就去動它，必然重演改 A 壞 B。**

所以這份藍圖的脊椎是：

> **對外，讓整座館是一次不中斷的呼吸；對內，讓每一次改動都有護欄，使這個有機的展演生命體能被「可控地」迭代。**

工程仍是手段，審美仍是目的。但這一輪工程的角色變了——它不是搭更多鷹架，它是一支**音叉**：負責讓全館的一致性可被校準、可被守住。

> **給未來的自己**：當你只剩工程的語言、開始把 E 系列當成「正事」、把 G 系列當成「之後再說的美化」時，停下來。順序剛好相反——**E 系列存在的唯一理由，是讓 G 系列能安全發生。** 護欄不是為了工程的整潔，是為了讓那一片黑能連成一體而不被你自己碰碎。

## 一、現況的兩個誠實觀察（基準線）

寫 v2 前，把讀過原始碼後的判斷記成白紙黑字。

### 1.1 一體性的斷點（感性）

| 斷點 | 證據 | 體感 |
|---|---|---|
| **三個影像入口並存** | 導覽 `作品 / 影像 / 系列` 是三個概念相近的入口（[Base.astro](src/layouts/Base.astro)）：作品＝單件 artwork 流、影像＝gallery 印樣捲、系列＝策展序列 | 觀者無法憑直覺知道該推哪扇門——這是檔案管理員的分類學洩漏到大廳 |
| **「共用一片黑」是宣言不是事實** | [room-veil.ts](src/scripts/room-veil.ts) 只在暗↔亮跨房時覆蓋淡出；同屬暗房的 作品→內頁→影像 之間是 ClientRouter 預設硬換頁 | 在同一個暗室裡走動，卻每走一步牆都重刷一次 |
| **顯影只發生在載入、不發生在移動** | 顯影（lqip→銳利）是暗房招牌手勢，但只綁在影像載入，換頁沒有它 | 換頁像點連結，不像沖洗下一張 |
| **空房沒有被編排** | series 索引空、themes 稀；空狀態文案是「檔案尚未建立」這種 App 語氣 | 推開一扇門進到空房，且那房沒有館的聲音 |

### 1.2 可控迭代的缺口（理性）

| 缺口 | 證據 | 後果 |
|---|---|---|
| **零自動護欄** | `package.json` 無任何 test 工具、無 CI、無視覺回歸 | 改 A 壞 B 全靠肉眼，跨頁才現形的 bug 必漏 |
| **VT 脆弱性是反覆同一類坑** | [components.css](src/styles/layers/components.css) 與記憶 [vt-grid-cjk-collapse](../../.claude/projects/-Users-taylor-Desktop-profolio/memory/vt-grid-cjk-collapse.md) 都記著「scoped 樣式 VT 後失效→下沉全域」 | 同一個坑踩過至少三次，沒成為規則就會有第四次 |
| **設計契約只是散文** | DESIGN §4.1 綠白名單、§8.2 token 債，全靠人記得 | 漂移必然累積，沒有東西在守 |
| **腳本生命週期各寫各的** | 七支腳本各自手寫 `astro:after-swap` 重綁（[reveal.ts](src/scripts/reveal.ts)） | 漏綁＝「改了沒套用到」 |
| **顯示面與內容沒有契約** | EXIF／series／GPS 顯示面建好卻休眠（DESIGN §8.2） | 「建好但休眠」冒充「完成」 |

# H0 ｜先立護欄

> 這個地平線一個「觀者看得到的功能」都不做。它純粹是讓接下來的一體性工作不會改 A 壞 B。三項都很便宜。

## E1 — 視覺回歸護欄（最高槓桿）

### 🎨 藝術／設計目標
護欄本身不浪漫，但它**保護的東西很浪漫**：整座館的視覺一致性。一個攝影師不會讓沖印機隨機改變顯影濃度——他要的是**可重現**。E1 就是這台沖印機的校準片：每次改動後，全館每一頁、每一房、每一個尺寸都會被比對一次，確保「這張照片今天沖出來，和昨天一樣」。它讓審美的一致性從「我希望沒壞」變成「我知道沒壞」。

### 🧭 UIUX 細節
- 覆蓋每種頁型：首頁(暗) / post(暗) / gallery(暗) / series(暗) / theme(暗) / sketch(暗) / writings(亮) / about(亮) / 404。
- 每頁 × 桌機(1440) + 手機(390)。
- **關鍵**：至少一條路徑是「用站內連結 **導覽過去**」而非 fresh load——因為 VT 塌陷這類 bug 只在 ClientRouter 換頁後現形，reload 看不到。
- 種子資料即可跑，不依賴真內容。

### 🔧 工程：根因／手段／驗收
- **根因**：所有驗證靠人工截圖 → 跨頁才出現的 bug 必漏。
- **手段**：`@playwright/test` + 視覺快照。`playwright.config` 起本地 `astro dev`（webServer），snapshot 全頁型矩陣，含一條導覽路徑。
- **驗收**：`npm run test:visual` 一鍵；像素 diff 超閾值即紅；baseline 入庫。

## E4 — 「完成」的定義（DoD・流程不是程式）

### 🎨 藝術／設計目標
「只做一半」對一座館是致命的——半亮的燈比全暗更糟，因為它讓人以為這裡有人照顧、其實沒有。DoD 是策展人收工前繞場一圈的紀律：燈都亮齊了嗎、每個房間都走過了嗎、從入口走進來（不是憑空出現在房裡）的動線順嗎。

### 🔧 工程：根因／手段／驗收
- **根因**：done 的標準只有「fresh load 看起來對」。
- **手段**：把 v1 §三那份「四問」commit checklist 補上工程 DoD 三條，寫進 PROGRESS 當硬規則。
- **驗收**：每筆落地備註都能回答這三條（見 [§五](#五給每一次改動的-dod)）。

## E6 — VT 脆弱性根治原則（把踩過三次的坑寫成法律）

### 🎨 藝術／設計目標
這座館用 view-transition 換頁是為了「連續」（DESIGN §9 光的樂章）。但連續的代價是一類隱蔽的塌陷——中文圖說直排、版面崩成一字寬。把根治原則寫成法律，等於宣告：**連續感不能用版面的穩定去換**，兩者都要。

### 🔧 工程：根因／手段／驗收
- **根因**：「碰 grid 軌道的版面若留在頁級 scoped `<style>`，VT 換頁後會塌」已被發現三次，卻沒成為規則。
- **手段**：明文架構守則——**任何觸及 grid 軌道／欄寬的版面樣式一律放全域 layer；頁級 `<style>` 只放不影響軌道的微調**。理想上由 E2 lint 掃出頁面 scoped 裡的 `grid-template`。
- **驗收**：守則寫定（DESIGN §14 / PROGRESS）；既有 gallery 已是範本；新版面遵循。

# H1 ｜一體性對外

> 護欄就位，現在可以放心動全域過場——讓整座館連成一個身體。

## G4 — 統一的過場語法：顯影即移動

### 🎨 藝術／設計目標
暗房的招牌手勢是**顯影**——模糊轉銳利，像顯影紙在藥水裡浮出影像。但它目前只發生在「影像載入」，不發生在「換頁」。讓**每一次站內導覽都是一次顯影**，整站的移動就有了單一的儀式語法：你不是在點連結，你是在沖洗下一張底片。這是把 zine「翻頁」的身體動作，誠實搬進數位載體。

### 🧭 UIUX 細節
- 用 Astro `transition:name` 為**封面影像**建立跨頁連續：首頁 entry 的封面 → post 頁的 cover，是**同一張在延續**，不是兩張各自淡入。觀者的眼睛被那張影像牽著走進下一頁。
- 過場時長／easing 全走既有 `--dur` / `--ease-out` token，**不新增任何動態方言**（DESIGN §4.5）。
- `prefers-reduced-motion`：退回無動畫直切，功能完整。
- 克制：只有封面影像帶連續，標題與 meta 維持安靜——影像永遠是被牽著走的主角。

### 🔧 工程：動哪些檔／驗收
- [Plate.astro](src/components/Plate.astro)：支援可選 `transitionName`。
- [index.astro](src/pages/index.astro)、[post/[slug].astro](src/pages/post/[slug].astro)、gallery／series 索引：對應封面共用 `transition:name`（以 slug 命名，保證來源／目的地配對唯一）。
- **依賴 E1**（這是改全域過場，最需要回歸護欄看著）。

## G2 — 連續的暗

### 🎨 藝術／設計目標
DESIGN §9 寫「一整座館共用同一片黑」，但工程上每個暗房頁是**各自獨立地黑**。應該讓同屬暗房的頁間**完全不洗掉黑底**——像在同一個暗室裡走動，燈光換了、牆沒換。連續的暗是這座館最基本的身體感：黑不是背景，黑是空間本身。

### 🧭 UIUX 細節
- 同房導覽不閃白：換頁瞬間 `html`/`body` 背景必須已經是該房表面色（暗房不該有任何一幀白閃）。
- veil 只在跨房時動（[room-veil.ts](src/scripts/room-veil.ts) `from===to` 已 return ✅）——確認此行為在 G4 過場下仍成立、不打架。
- 評估（不強求）：3D 光場的「環境殘光」是否以極低成本延續到圖錄頁背後（DESIGN §9 第二樂章）。**ROI 不明時不做**——但連續的黑底是底線，必做。

### 🔧 工程：動哪些檔／驗收
- 確保根層背景 = `--surface`（隨 `data-room` 重映），且在 ClientRouter 換頁的兩幀之間不回退。
- **驗收**：E1 的導覽路徑快照在暗房間切換時無白閃幀。

## G1 — 三個影像入口的釐清（最大的直覺手術）

### 🎨 藝術／設計目標
`作品 / 影像 / 系列` 三個入口，是檔案管理員的分類學洩漏到大廳。v1 已用同樣理由把「主題」降為 facet——同一把刀該再揮一次。但三者**各有真實的策展意義**，不該粗暴合併：
- **作品**＝全部，時間軸式的檔案瀏覽。
- **影像（gallery）**＝一捲的兩種看法（印樣＋放映）。
- **系列（series）**＝藝術家的手，我決定你以什麼順序翻閱。

問題不在它們存在，在於**觀者站在大廳時，無從知道每扇門後是什麼**。

### 🧭 UIUX 細節（決策：給框架，不收斂）
- **每個索引頁頂部一句策展短語**，告訴觀者「你會在這裡看到什麼」——把分類學翻譯成館的語言（呼應 v1 H0.1 主題頁的「以此凝視」改寫）。
- **三者互相導流**：不是三份平行空目錄，而是同一座館的三個展廳，彼此看得見對方。
- 與 G3 協作：**導向空房的入口，在無內容時隱藏**（series 現在空 → 暫不在導覽佔位），避免把人導進空房。
- 資料模型一行不動（可逆，同 v1 H0.1）。

### 🔧 工程：動哪些檔／驗收
- gallery／series／writings 索引頁：加策展短語標頭 + 交叉連結。
- [Base.astro](src/layouts/Base.astro)：空房入口條件隱藏（接 G3）。
- **依賴 E1 + G3**。

# H2 ｜契約化 + 空房的尊嚴

## E2 — 設計契約的可執行化（讓 DESIGN.md 從散文變成機器守則）

### 🎨 藝術／設計目標
綠是全館唯一的生命訊號，它的力量來自**稀缺與一致**。但稀缺只靠「我記得別亂用綠」是守不住的——總有一次手滑。把白名單變成 lint，等於請一台機器替你守住那份克制，讓「綠是稀缺的生命訊號」這句話**永遠為真**，而不是「大部分時候為真」。

### 🔧 工程：根因／手段／驗收
- **根因**：DESIGN §4.1 白名單、§8.2 token 債全是散文，沒有東西在守。
- **手段**：`stylelint` 規則——①`tokens.css` 以外禁裸 hex ②綠只能來自 `--accent`/`--life*` ③字級／間距裸值警告（對齊 scale）④頁級 scoped 禁 `grid-template`（接 E6）。
- **驗收**：`npm run lint:design` 綠；新違規可被擋。

## E3 — 行為腳本生命週期契約

### 🎨 藝術／設計目標
這座館的「活」——準星跟著游標、綠光脈動、聲音開關、顯影——全靠這七支腳本。當其中一個在換頁後忘了重新醒來，館就**局部死掉**一塊，而且是靜默地死。統一的生命週期契約，是確保**每一次換頁後，館的每一個活著的部位都重新呼吸**。

### 🔧 工程：根因／手段／驗收
- **根因**：七支腳本各自手寫 `astro:after-swap` 重綁，模式不一 → 漏綁＝「改了沒套用到」。
- **手段**：抽共用 `onPage(fn)`（首次執行 + ClientRouter 重綁 + 換頁前清理），七支統一改用。生命週期只寫一次。
- **驗收**：每支腳本走同一契約；E1 的導覽路徑快照覆蓋「換頁後行為重綁」。

## G3 — 空房間的尊嚴

### 🎨 藝術／設計目標
在藝術空間裡，**空不該是壞掉，空該是被編排的靜默**。「檔案尚未建立」是 App 的語氣，不是這座館的聲音。一個還沒沖洗的暗房，應該說「這捲還沒沖洗」，而不是丟一個錯誤狀態給你看。

### 🧭 UIUX 細節
- 空狀態文案全部改寫為館的語氣（暗房用顯影／底片的隱喻，亮房用書房／未寫的隱喻）。
- **導向空房的導覽項，在無內容時隱藏**——別讓人推開一扇門進到空房（誠實 IA，呼應 v1 主題降級與 G1）。

### 🔧 工程：動哪些檔／驗收
- 各索引頁空狀態分支文案；[Base.astro](src/layouts/Base.astro) 依內容計數條件渲染導覽項。

## G5 — 綠的一致性審計

### 🎨 藝術／設計目標
綠的可信度取決於**到處都一致**。一個地方的綠偏黃、另一個地方該綠卻沒綠，整套「生命訊號」的隱喻就鬆了。

### 🧭 UIUX 細節
- 對照 DESIGN §4.1 白名單逐一核對：focus-visible 環、aria-current、進度條填充、reticle 括弧、wip 脈動、內文連結下緣、sidenote 記號——是否都真用 `--accent`、且暗／亮房亮度版本正確。

### 🔧 工程：動哪些檔／驗收

# H3 ｜候選（預設懷疑・ROI 逐項驗）

## 五、給每一次改動的 DoD

動手前問 v1 的**四問**（任一答「否」就停）：

1. 它讓某一張照片更被看見嗎？
2. 它增加了上稿摩擦，還是減少了？
3. 綠色用對地方了嗎？（DESIGN §4.1 白名單）
4. 影像的存在感仍高於這個新元素嗎？

收工前過 v2 的**三條**（這三條專治「只做一半 / 改了沒套用」）：

5. **兩房都看過了嗎？**（暗房 + 亮房）
6. **手機都看過了嗎？**（390 寬不只桌機）
7. **是「導覽過去」再看的，不只是 reload？**（VT 坑只在這裡現形）
