## 一、概念 — 這個空間是什麼

這是一座**會衰敗的美術館**。它不是組合庫（portfolio），它是一個攝影師凝視光的地方。

空間的氣質來自三種並存的狀態：

- **光**——它存在、它消失、它留下痕跡。
- **衰敗**——事物在緩慢瓦解，這不是悲觀，而是時間的誠實。
- **生機**——在廢墟中還有一粒東西仍在呼吸。那粒東西是綠色的。

整個設計系統由這個意象驅動。灰階不是「中性底色」的技術選擇，而是敘事：**當一切退成灰，你才能看見什麼還活著**。

## 二、設計框架 — 兩個房間，一條脊椎

### 空間結構

網站由兩種截然相反的表面組成，共享同一套結構骨架：

| 房間 | 內容 | 表面 | 光線 | 氣質 |
|---|---|---|---|---|
| **暗房** | 首頁 / 作品 / 圖錄 / 3D / 互動作品 | `--gallery #0c0c0b` | 暗場，影像像投影牆上的發光體 | 美術館夜間觀展，凝視 |
| **亮房** | 散文 / 閱讀 / About | `--paper #f4f4f2` | 自然紙底 | 書房閱讀，思考 |

這個切換不是「深色模式 / 淺色模式」的使用者設定——它是**空間敘事**：走進長文＝走進唯一亮著燈的房間，這個「點亮」本身是意義。

### 脊椎（兩房共用）

- **12欄網格** + 流動外緣（`--margin`）+ 統一間距系統
- **同一套字體**（無襯線介面 / 宋體長文）
- **同一套 token**（色彩、間距、字級、動態）
- **同一條綠**（地衣綠，在兩個表面各有一個亮度版本）

## 三、設計脈絡 — 藝術參照如何變成設計決策

這個網站有意識地繼承四種當代藝術脈絡。每種脈絡對應一層視覺決策，不是裝飾，而是世界觀：

### 層 1：資料精度（Ikeda / Anadol / Reas）
**藝術問題**：資料本身是否可以有美學？
**設計決策**：每件作品的右下角有一條等寬字「資料條」——決定性的座標、編碼、頻道。這些值由 slug 種子生成，或接入真實 EXIF/GPS。視覺語言：`--font-mono`、tabular-nums、極小字級。綠色 `__sig` 欄位是資料的「活訊號」。

### 層 2：光的環境（Turrell / Eliasson / Janssens）
**藝術問題**：光本身能否成為空間的建築材料？
**設計決策**：3D 光點雲（hero）不只是裝飾。它的密度、溫度、阻尼感刻意模仿 Turrell 光裝置的「沉思感」。光場隨捲動消融——不是消失，而是稀釋成圖錄背後的環境殘光，「一整座館共用同一片黑」。

### 層 3：凝視與監視（Steyerl / Lozano-Hemmer / Paglen）
**藝術問題**：誰在看誰？
**設計決策**：暗房（圖錄）專屬的機器視覺準星（reticle），跟隨游標、鎖定作品時四角括弧框選。這個元素不在亮房出現——閱讀不被監視。閱讀是私密的，圖錄是被凝視的。

### 層 4：網路藝術的反應性（JODI / Rozendaal）
**藝術問題**：介面本身能否成為作品的一部分？
**設計決策**：準星的反應手勢、一粒綠光的慢脈動、快門音響應的鎖定——這些都讓網站「像一台在運作的儀器」，而不只是靜態的展示台。

## 四、視覺語言系統

### 4.1 色彩哲學

**唯一原則：彩度即訊號，灰階即空間。**

彩色（綠）是系統裡「稀缺的生命訊號」，不是裝飾色。用量克制的程度直接決定「生機」是否廉價——綠用多了，隱喻就死了。

```
亮房灰階                          暗房灰階
──────────────────────────────    ─────────────────────────────
--paper  #f4f4f2  近美術館白      --gallery     #0c0c0b  暗室
--ink    #0e0e0d  主文字          --gallery-ink #ececea  暗場文字
--ink-70          次要文字         --g-ink-70             次要文字
--ink-45          輔助文字         --g-ink-45             輔助文字
--ink-15          髮絲線           --g-ink-20             最弱輔助
                                  --hairline-dark         極淡分隔線

唯一彩色（兩種亮度適應兩個表面）
──────────────────────────────
--life      #4f5e44   地衣綠（亮底）
--life-dark #93a47f   地衣綠（暗底）
```

**為什麼是「地衣綠」？** 不是草地的鮮綠，不是網頁的青綠。地衣長在石頭上、在廢墟的縫隙裡——它就是「在衰敗中生長的東西」的顏色。這個選色本身就是敘事。

**綠色白名單（tier 2 — 「可被啟動的訊號」）**

| 使用場景 | 理由 |
|---|---|
| `aria-current` 當前導覽 | 你在這裡 |
| `:focus-visible` 聚焦環 | 被觸碰的瞬間 |
| 閱讀進度條填充 | 進行中的狀態 |
| WIP 作品標記（慢呼吸動畫） | 還在呼吸的作品 |
| 內文連結下緣細線 + hover 轉綠 | 可互動的文字 |
| hover 時：nav 下緣、圖錄標題、meta 連結 | 觸碰的生機 |
| 3D 光場中的一粒綠光（慢脈動） | 整座館唯一的「活點」 |
| 機器視覺準星（reticle）括弧 | 凝視的訊號 |
| CV 時間軸最新節點（慢呼吸） | 當下的生命跡象 |

**仍不准**：任何背景填色、按鈕底色、大色塊、裝飾性色塊。

### 4.2 字體系統

**核心原則：字體是空間材質，不只是文字載體。**

兩套字體，各自服務一個房間的氣質：

```css
--font-read: 'Noto Serif TC', 'Source Han Serif TC', 'Songti TC', Georgia, serif;
/* 亮房長文。宋體有文學的重量與溫度，讓閱讀者沉進去。*/

--font-ui:   'Helvetica Neue', Helvetica, 'Inter', Arial, 'PingFang TC', sans-serif;
/* 介面預設。瑞士無襯線：中性、精準、不干擾內容。*/

--font-mono: 'SF Mono', 'JetBrains Mono', ui-monospace, monospace;
/* 資料精度層。等寬字的機器感，承載座標、編碼、時間戳。*/
```

**字重階梯（依房間混用，不是平階層）**

| 使用情境 | 字重 | 邏輯 |
|---|---|---|
| 暗房大標（作品名、圖錄條目） | 400 Regular | 展覽標籤的優雅：靠尺寸與字距，不靠粗細 |
| 亮房標題（散文、About） | 600 Semibold | 長文裡清晰的段落錨點 |
| 長文內文（宋體） | 400 Regular | 呼吸、透氣 |
| 介面（導覽、meta、標籤） | 500 Medium | 中性、不突出 |

**圓角：`--radius: 0px`（全站直角）**。刻意的——藝術品和建築空間是硬邊的，圓角會把氣質往「網頁 App」拉。

---

### 4.3 字級系統（modular scale 1.25）

```
--step--2: 0.72rem   micro label（資料條、編號）
--step--1: 0.82rem   small UI / figcaption
--step-0:  1rem      UI 基準（17px root）
--step-1:  1.25rem   中型介面
--step-2:  1.5625rem
--step-3:  1.953rem
--display: clamp(1.7rem, 3.6vw, 3rem)   頁面大標
--hero-title: clamp(1.6rem, 3.4vw, 2.7rem)
--read-size: 1.12rem（≈19px）   亮房長文基準
```

標題尺度刻意保守（max 3rem）——不爭奪視覺焦點的宏大感，讓**影像的存在感高於標題的存在感**。

---

### 4.4 間距系統（8px 模數）

```
3xs:4  2xs:8  xs:12  sm:16  md:24  lg:32  xl:48  2xl:64  3xl:96  4xl:128  5xl:160
```

流動外緣：
```css
--margin:  clamp(20px, 5vw, 90px)   /* 行動 20px / 桌機最大 90px */
--col-gap: clamp(16px, 2.2vw, 30px)
--measure: 40rem                     /* 長文閱讀寬度（約 38-40 中文字）*/
```

**無卡片、無陰影、無漸層。** 影像直接落在表面上，不加框。卡片會讓作品「被包裝」；影像應該直接呼吸在空間裡。

---

### 4.5 動態語言

動態服務「沉思」，不爭奪注意力。

| 用途 | 時長 | Easing |
|---|---|---|
| 入場揭示（opacity + translateY） | 1.5s（`--dur-slow`） | `cubic-bezier(0.16, 1, 0.3, 1)` |
| 影像顯影（lqip → 銳利） | 1.5s | 同上 |
| hover / 狀態變化 | 0.4–0.6s | `ease` |
| 房間切換（光揭示） | 0.6s | `ease` |
| WIP / CV 節點慢呼吸 | 3.6s | 無限循環 |

`prefers-reduced-motion`：停止所有動畫，保留靜態視覺。這是無障礙設計也是美學選擇——靜止的光場仍然完整。

## 五、影像哲學

**影像永不裁切**（除索引縮圖）。裁切是對影像構圖的暴力。作品的邊界是攝影師決定的，不是版面。

- 作品封面：置中 + 四周大留白，像掛在暗牆上的單一印樣。
- 影像集（Gallery）：3:2 印樣格（contact sheet），「影像＝複數的一捲」。
- 放映模式（Lightbox）：滿版黑場，序列翻閱，顯影轉場。

**顯影隱喻**：所有影像先顯示 LQIP（低解析底圖），載入後 crossfade 顯現——模擬暗房顯影紙從模糊轉銳利的過程。這不是技術妥協，而是刻意的敘事。

## 六、閱讀室規格（亮房長文）

對標：報導者、Patreon 的長文體驗。

**硬指標**：
- 內文 `--read-size`（≈19px）/ 行高 1.85 / measure 40rem（約 680px）
- 段距 `> * + * { margin-top: 1.6em }`
- 向左對齊（非 justified）：中英混排下 justified 會撐出不規則字距
- 標頭靠左 + 下緣髮絲（報導者式，不置中）

**編輯工具箱**：
1. **文章頭部**：大標 → 導言（lead，略大、實墨） → meta 列（日期·閱讀時間·分類）
2. **閱讀進度條**：頂部髮絲、綠色填充（合法綠用法）
3. **抽言（pull quote）**：大宋體、最少引號、左右大留白
4. **圖說系統**：`figcaption` 帶編號、無襯線小字、含標題/來源欄位
5. **旁註（sidenote）**：桌機（≥1180px）浮出 measure 右側、手機改 inline 縮排左線
6. **章節記號**：`.prose hr` 置中短髮絲

不用 drop cap（與克制衝突）。開場重量交給導言段落承擔。

## 七、響應式骨架

**四個具名斷點（不新增第五個）**：

| 名稱 | 值 | 意義 |
|---|---|---|
| `sm` | 560px | 手機 → 寬手機；header 收回單列 |
| `md` | 720px | 單欄 → 雙欄；側欄 sticky |
| `lg` | 1180px | 旁註浮出頁邊 |
| `xl` | 1440px | 內容最大寬 |

**螢幕方向**：
- 直幅（預設）：閱讀/索引主軸
- 橫幅手機（`landscape + max-height: 540px`）：影像優先，chrome 收最小

**觸控命中區**：coarse pointer 所有可點元素 ≥ 44×44px（透明 padding 擴張，不改視覺尺寸）。

### 8.2 尚未成熟的部分

| 面向 | 評估 | 問題 |
|---|---|---|
| **間距 token 落實** | ⚠️ 部分 | `components.css` 仍有散落的 `clamp(2.2rem, 6vh, ...)` 等手寫值，未走 `--space-*` token |
| **字級 token 落實** | ⚠️ 部分 | `.work__meta { font-size: 0.78rem }` 等硬寫值未對齊 scale（低優維護債） |

| **斷點一致性** | ⚠️ 部分 | `@media (min-width: 720px)` 已對齊 `md`，但部分頁級 `<style>` 仍有未遷移的舊值 |
| **無障礙審計** | ⚠️ 部分 | focus trap 在 Lightbox 完成；其他元件的鍵盤/ARIA 未系統性審計 |
| **v3.1 藝術融合尾巴** | ⚠️ 候選 | 掃描線/網格（Ikeda）、Rozendaal 著陸——已列入候選但未開始 |

### 8.3 系統性結論

這個設計系統的**架構成熟度**在當代藝術個人網站裡是少見的高——token 地基、層級系統（CSS @layer）、兩房結構、藝術參照的有意識融合都已落地。**當前真正的風險不是設計系統本身，而是「骨架比肉身豐富」**：工程的地基比內容填充走得更快。

> **2026-06-27 更新**：[ROADMAP](ROADMAP.md) 的 H0／H1 收斂與通電已執行——主題降為 facet、放映加 Grid 總覽、真 EXIF 讀檔、OG 立面、論述面板、Series 連續放映皆落地。這讓上面的結論更尖銳而非過時：**儀器又添了幾件，但它們大多在空轉**（EXIF／series 等著真內容點亮）。所以方向不變，且第 1 點的優先級更高了。

接下來的設計工作方向：
1. 把已建好的結構用真實內容填滿（**最高優先**：真照片＋EXIF、建立 series）
2. 修幾個局部的 token 一致性問題（低優維護債）
3. 選擇性推進 v3.1 候選（只做能讓作品「更被看見」的，拒絕功能劇場）；地圖（GPS）維持 HOLD

## 九、3D 光場（沉思內核）

維持已調好的密實柔光點雲（Turrell / Anadol 沉思感）：

**三個樂章**：
1. **Hero（0–100vh）**：光場沉降、後退、淡出、微縮
2. **沉入圖錄**：光場不歸零，稀釋成圖錄背後的環境殘光（「一整座館共用同一片黑」）
3. **指標互動**：極輕微聚散、重阻尼、位移極小

**侵蝕曲面**：獨立的傾斜細線網格地面，FBM 噪聲頂點位移，灰階極淡。「衰敗」由此元素承載，光雲（生機/沉思）原貌保留。綠仍只在那一粒活光點。

**綠霧（haze）**：3D 光場後墊一層極淡 additive 綠色 plane（Janssens 彩色霧），隨捲動淡出。
